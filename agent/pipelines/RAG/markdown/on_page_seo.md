# On-Page SEO: Comprehensive Guidelines and Best Practices

## 1. Title Tags
Title tags are one of the most critical on-page ranking factors. They tell users and search engines what the page is about.
- **Length Threshold**: Keep titles between 50-60 characters (or roughly 600 pixels) to prevent truncation in SERPs.
- **Keyword Placement**: Place the primary target keyword as close to the beginning of the title as possible.
- **Brand Inclusion**: Append the brand name at the end (e.g., `Primary Keyword - Secondary Keyword | Brand Name`).
- **Uniqueness**: Every single page must have a 100% unique title tag.
- **Click-Through Rate (CTR) Optimization**: Use modifiers like "Best", "Guide", "2024", or actionable verbs to improve CTR.
- **Avoid Keyword Stuffing**: Do not list variations of the same keyword (e.g., `Cheap Shoes, Buy Shoes, Discount Shoes`).

### Example Title Tag Configurations:
- **E-commerce Product**: `[Product Name] - Buy Online | [Brand]`
- **Blog Post**: `How to [Action]: A Complete Guide for [Year] | [Brand]`
- **Local Service**: `[Service] in [City, State] | [Company Name]`

## 2. Meta Descriptions
While not a direct ranking factor, meta descriptions directly impact organic CTR, which is a behavioral signal.
- **Length Threshold**: Keep between 150-160 characters (desktop) and around 120 characters (mobile).
- **Intent Matching**: Accurately summarize the page content matching the user's search intent.
- **Call to Action (CTA)**: Include a soft CTA (e.g., "Learn more", "Get a free quote", "Read our guide").
- **Keyword Formatting**: Include the primary keyword. Google bolds exact matches in the SERP, drawing the user's eye.
- **Uniqueness**: Avoid auto-generated or duplicate meta descriptions across multiple pages.

## 3. Heading Structure (H1, H2, H3, H4)
Headings establish a clear hierarchical structure for search engine crawlers and improve readability for users.
- **H1 Tag**:
  - **Rule**: Strictly ONE H1 tag per page.
  - **Content**: Should closely align with the Title Tag but can be slightly more descriptive.
  - **Keyword**: Must contain the primary keyword.
- **H2 Tags**:
  - Use to break down the main content into broad subtopics.
  - Excellent place to naturally include secondary keywords or semantic variations (LSI).
- **H3 and H4 Tags**:
  - Use for nested sub-points under H2s.
  - Highly effective for formatting content for Featured Snippets (especially lists or step-by-step guides).
- **Hierarchy Rule**: Never skip heading levels (e.g., jumping from H1 directly to H3).

## 4. URL Structure
URLs should be clean, descriptive, and logically structured.
- **Length**: Keep them as short as possible. A study showed URLs ranking on page 1 average ~59 characters.
- **Format**: Use strictly lowercase letters.
- **Separators**: Use hyphens (`-`) to separate words. NEVER use underscores (`_`) or spaces.
- **Keyword Inclusion**: Include the exact primary keyword.
- **Stop Words**: Remove unnecessary stop words (a, and, the, of) to keep the URL concise.
- **Directory Structure**: Keep it close to the root domain. `domain.com/category/post-title` is better than `domain.com/folder1/folder2/folder3/post-title`.

## 5. Image Optimization and Alt Tags
Images provide context to search engines and are essential for Google Image Search.
- **Alt Text (Alternative Text)**:
  - **Rule**: Every functional or content-bearing image must have descriptive alt text.
  - **Best Practice**: Describe the image specifically as if describing it to a blind person.
  - **Keyword**: Include the target keyword ONLY if it naturally describes the image. Do not keyword stuff alt tags.
- **File Naming**: Name files descriptively before uploading (e.g., `red-running-shoes.jpg`, NOT `IMG_4921.jpg`).
- **File Format & Compression**: Use modern formats like WebP. Compress all images (target <100KB for standard images, <300KB for heroes) without losing visual fidelity.
- **Image Dimensions**: Explicitly specify `width` and `height` attributes to prevent Cumulative Layout Shift (CLS).

## 6. Internal Linking Rules
Internal links pass "link equity" (PageRank) around the site and help crawlers discover new pages.
- **Anchor Text**: Use descriptive, keyword-rich anchor text. Unlike external backlinks, exact-match anchor text is highly encouraged for internal linking.
- **Contextual Placement**: Links placed high up in the main body text pass more SEO value than links in the footer or sidebar.
- **Hierarchical Linking (Siloing)**:
  - Link from top-level category pages down to sub-categories.
  - Link from blog posts up to relevant pillar pages or product/service pages.
- **Orphan Pages**: Ensure zero pages exist without at least one internal link pointing to them. A page with no internal links is an "orphan" and is rarely indexed or ranked.
- **Link Volume**: Avoid overwhelming pages with internal links. Keep it reasonable and user-focused.

## 7. Keyword Placement Best Practices
Strategic keyword placement reinforces topical relevance without triggering spam filters.
- **First 100 Words**: Place the primary keyword within the first 100-150 words of the body content. This establishes immediate context.
- **Natural Density**: Avoid specific "keyword density" percentages, but generally aim for 1-2% for the primary keyword. Read it aloud; if it sounds robotic, reduce the keyword usage.
- **Semantic Keywords (LSI)**: Use variations and conceptually related terms. If writing about "SEO", use terms like "search engines", "ranking factors", "crawling", "indexing".
- **URL, H1, Title**: The "Holy Trinity" of keyword placement. The exact match primary keyword must be in all three.
- **Conclusion/Summary**: Reiterate the main keyword in the concluding paragraph.

## 8. Entity SEO and Schema Intersections
On-page SEO in 2024 heavily relies on understanding "Entities" (people, places, concepts).
- **Clarity of Entity**: Ensure the page clearly defines the primary entity. Using "What is [Entity]?" style H2s helps search engines map the page to their Knowledge Graph.
- **Co-occurrence**: Mentioning related entities strengthens the primary entity signal (e.g., an article about "Steve Jobs" MUST mention "Apple", "Pixar", "iPhone").
